# AI 시장검증 플랫폼 (가칭)

> 아직 유저가 없는 웹 서비스를, 출시 전에 가상 고객 수백 명에게 미리 물어보는 AI 시장검증 플랫폼

## 1. 문제 정의

실제 서비스를 구현하는데에 있어서 시장조사는 필수적이다 하지만 시장조사에 대한 기업의 입맛대로 데이터를 얻는 과정은 돈이 많이 들고 과정도 번거롭다 
하지만 대신해주는 가상 인간 가상 페르소나가 있으면 좋지않을까? 마음대로 중간에 질문지를 바꾸어도 좋고 묻기 어려운 질문이여도 좋다

실제 시장조사: 수백만 원 + 수 주 소요
-> 가상 페르소나 서비스를 사용하면 쉽고 돈을 조금 내고도 리서치가 가능하다

이 플랫폼은 실제 고객을 만나기 전" 단계의 검증 공백을 채운다.

## 2. 핵심 기술 — 페르소나 디커플링 엔진

LLM에게 단순히 "고객 100명 만들어줘" 하면 통계가 왜곡되고 응답이 동질화된다. 이를 막기 위해 역할을 분리한다.

| 원칙 | 내용 |
|---|---|
| 숫자는 데이터가 | 시장 분포(통계청 인구 분포 등)에서 집단별 비율과 수치 축(가격민감도·혁신수용도)을 고정 |
| 이야기는 LLM이 | 고정된 수치를 조건으로 개인 서사만 생성 (2단계 생성) |
| 동질화 방지 | 페르소나 간 독립 컨텍스트에서 병렬 실행 |
| 집계 가능 | 응답을 구조화 스키마(반응점수·구매의향·반발태그)로 강제 → "수용률 34%" 같은 수치 산출 |

이 엔진으로 범위가 다른 2종의 페르소나를 생성한다.

- **타겟 페르소나**: 실사용자층, 좁은 범위 → 서비스 검증용
- **다중 페르소나**: 시장 전체, 대규모 → AI 시장 검증용

## 3. 주요 기능 (5개 메뉴)

### ① 시장 조사 — 검증의 재료 만들기
서비스 개요를 등록하면 자동 분석 후 리포트 생성.
- 트렌드 요약 / 시장 브리핑 / 경쟁 서비스 분석 / 타겟 고객 분석(세그먼트 도출)
- 각 섹션 끝에 "다음 검증 액션" 시사점 제공
- 도출된 세그먼트 → 타겟 페르소나 생성의 입력값

### ② 서비스 검증 — "타겟에게 통하는가"
- **컨셉 테스트**: 세그먼트별 반응(긍정/부정·대표 인용·반발 포인트), 오해율, 시사점
- **BM 검증**: BM 캔버스 9칸 자동 평가(강/보통/약 신호등) + 약점별 검증 액션 연결, 요금안 노출 → 수용률·최대지불가 수요곡선
- **기능 검증**: 강제 트레이드오프로 필수/좋음/무관심 분포 → 개발 우선순위
- 수정 후 재실행 시 전후 비교 ("반발 62%→31%")
- 산출물: 서비스 적합도 리포트 + AI 리뷰 패널(페르소나 심층 인터뷰)

### ③ AI 시장 검증 — "출시 후를 미리 돌려보기"
- 다중 페르소나 시뮬레이션 + 스트레스 테스트(블랙컨슈머 등 극단 페르소나 주입)
- AI 패널 집단조사(FGI)·토론(FGD): 페르소나 상호작용으로 반박 논리 포착
- 시장 반응 예측: 시계열 State 기반 이탈 원인 랭킹·개선안별 복귀의향 예측
- 산출물: 시장 검증 리포트 + AI 리뷰 패널

### ④ 마케팅 제작지원 — 검증 결과를 실행물로
- 반응 좋았던 메시지·세그먼트 기반 홍보 포스터·마케팅 콘텐츠 생성
- 카피 A/B를 가상 고객에게 사전 테스트 → 출시 전략 리포트

### ⑤ 프로젝트 관리 — 반복 검증의 기반
- 페르소나 세트 저장·재사용 (동일 집단 재검증으로 전후 비교 성립)
- 리포트 아카이브 + 전체 통합 GTM 전략 리포트 생성
